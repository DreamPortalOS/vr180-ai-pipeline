# 产品体验设计 — VR180 FPV 飞行体验

## 产品愿景

将 AI 生成的 2D 视频转化为沉浸式 VR180 第一人称飞行体验，
让用户在 VR 头显中感受身临其境的飞行快感。

### 核心体验

- 🎮 **FPV 第一人称视角** — 像真正在驾驶无人机飞行
- 🥽 **VR180 立体沉浸** — 在 Quest/Vision Pro 上体验真实深度感
- ✨ **AI 生成内容** — 无需实拍，通过提示词创造任何场景
- ⚡ **一键转换** — 2D→VR180 全自动流水线

## 内容制作流程（端到端）

### Step 1: 选择场景模板

从以下预设计模板中选择，或自定义提示词：

| 模板 | 难度 | 立体效果 | 推荐理由 |
|------|:----:|:--------:|----------|
| 🏔️ 峡谷飞行 | ⭐ | ★★★★★ | 强烈深度对比，最佳起始点 |
| 🌲 森林小径 | ⭐⭐ | ★★★★ | 树干提供良好视差 |
| 🌆 未来城市 | ⭐⭐ | ★★★★ | 建筑线条清晰 |
| ☁️ 云海日出 | ⭐ | ★★★ | 大气效果，柔和深度 |
| 🌋 火山飞行 | ⭐⭐⭐ | ★★★★★ | 戏剧性场景 |
| 🌊 海边悬崖 | ⭐⭐ | ★★★★ | 广阔空间感 |

### Step 2: 用 Gemini Omni 生成 2D 视频

在 `gemini.google.com` 使用 Omni Flash 模型：
1. 选择 **Text to Video** 模式
2. 从模板中选择提示词，或参考 `docs/prompt-design-guide.md` 自定义
3. 设定 10秒, 16:9, 1080p
4. 点击生成
5. 等待 1-2 分钟（Omni Flash 生成时间）
6. 预览效果，如果不满意可对话式编辑
7. 下载 MP4

### Step 3: 运行 VR180 转换流水线

```bash
# 激活虚拟环境
source .venv/bin/activate

# 完整流水线
python scripts/run_pipeline.py -i canyon_flight.mp4 -o canyon_vr180.mp4

# 更高质量（用 Base 模型）
python scripts/run_pipeline.py -i canyon_flight.mp4 -o canyon_vr180_hq.mp4 \
    --model-size base --codec h265 --crf 20
```

### Step 4: 在 VR 头显中播放

**Meta Quest:**
1. 通过 USB 传输 VR180 文件到 `Quest/Downloads/`
2. 使用 Skybox VR Player 或默认播放器
3. 视频会自动识别为 VR180 格式

**Apple Vision Pro:**
1. 通过 AirDrop 或文件应用传输
2. 使用 Moon Player 或其他 VR 播放器

**YouTube VR:**
1. 上传到 YouTube
2. YouTube 自动识别 VR180 元数据
3. 在 YouTube VR 应用中观看

## 用户体验质量目标

| 维度 | 最低可接受 | 目标 | 理想 |
|------|-----------|------|------|
| 立体感 | 有明显深度 | 物体凸出屏幕 | 沉浸式 3D 体验 |
| 深度稳定性 | 轻微闪烁 | 稳定无闪烁 | 完美时域一致性 |
| 分辨率 | 1920×960 (SBS) | 3840×1920 (SBS) | 7680×3840 (SBS) |
| 帧率 | 24fps | 30fps | 60fps |
| 编码 | H.264 | H.265 | H.265 10-bit |
| 延迟(总流程) | 10分钟 | 5分钟 | 3分钟 |
