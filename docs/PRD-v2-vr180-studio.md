# VR180 Studio — 产品需求文档 (PRD) v2.0

> **项目名称**: VR180 Studio
> **版本**: v2.0
> **日期**: 2026-06-22
> **状态**: DRAFT — 待确认

---

## 1. 产品愿景

**一句话**: 一站式 VR180 内容创作平台 —— 从 AI 视频生成到高质量 VR180 成片的全流程工作流。

**核心价值**:
- 非技术用户可在 Web 界面完成全部 VR180 制作流程
- 每一步可预览、微调、确认后再进入下一步
- 支持多种 AI 视频生成引擎（Seedance / Kling / Wan2.2 / Google Omni 等）
- 最终输出单眼 4K-8K 的 VR180 视频，可在 Quest/YouTube VR 上播放

---

## 2. 用户画像

| 角色 | 描述 | 需求 |
|------|------|------|
| **VR 内容创作者** | 有 VR 头显，想自己制作 VR180 内容 | 简单易用、高质量输出 |
| **独立电影人** | 想尝试沉浸式影片 | 分镜控制、配乐、色彩管理 |
| **技术开发者** | 想集成到自己的工作流 | API 接口、CLI 工具 |

## 4. Web UI 设计

### 4.1 技术栈

| 层 | 技术选型 | 理由 |
|---|---------|------|
| **前端** | React + TypeScript + TailwindCSS | 组件化、生态丰富 |
| **状态管理** | Zustand | 轻量、适合工作流状态 |
| **工作流引擎** | 自研 (参考 ComfyUI 节点图) | 支持 DAG 执行、断点续传 |
| **实时预览** | WebSocket + HLS | 流式预览视频 |
| **3D 预览** | Three.js + WebXR | VR180 实时预览 |
| **后端** | FastAPI (Python) | 与现有 pipeline 无缝集成 |
| **任务队列** | Celery + Redis | 异步长任务 |
| **存储** | 本地文件系统 + SQLite | 简单部署 |

## 6. 球面补全方案详细设计

### 问题分析

当前 pipeline 的等距柱状投影输出中，源画面（70° HFOV）仅占据球面中间约 38.9% 的垂直区域。上下区域为黑色：

```
┌───────────────────────────────┐
│ ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ │ ← 黑色 (~29%)
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │ ← 源内容 (~41%)
│ ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ │ ← 黑色 (~29%)
└───────────────────────────────┘
```

### 方案对比

| 方案 | 质量 | 速度 | GPU需求 | 推荐场景 |
|------|------|------|---------|---------|
| **A: AI Outpainting** | ⭐⭐⭐⭐⭐ | 慢(5-30s/帧) | 8GB+ VRAM | 最终产出 |
| **B: 渐变填充** | ⭐⭐⭐ | 极快(<0.1s) | 无 | 快速预览 |
| **C: 环境贴图** | ⭐⭐⭐⭐ | 快(<1s) | 无 | 室外场景 |
| **D: VR180标准** | ⭐⭐⭐⭐⭐ | N/A | 无 | 已有VR180规范 |

### 方案 D 说明 (推荐默认)

VR180 行业标准本身就是**不需要完整球面覆盖**的。YouTube VR、Meta Quest 等平台对 VR180 的定义是：

> VR180 = 只覆盖前方 180° 的立体视频，不需要后方和上下覆盖

因此 **黑色填充是正确的 VR180 行为**。播放器会自动处理上下区域（通常是纯黑或根据用户头部追踪调整）。

**建议**:
- **默认**: 保持当前黑色填充 (符合 VR180 标准)
- **可选**: AI Outpainting 作为高级功能（适合想要完整 360° 球面的用户）
- **预览**: 渐变填充用于快速预览

### 方案 A 实现思路 (AI Outpainting)

```python
class SphericalOutpainter:
    """将 VR180 半球面补全为完整 360° 球面。"""

    def __init__(self, model="sdxl-inpaint"):
        self.model = model

    def complete_sphere(self, equirect_frame, depth_map=None):
        """
        输入: 半球面等距柱状帧 (中间有内容, 上下黑色)
        输出: 完整球面等距柱状帧

        步骤:
        1. 识别内容区域和待补全区域
        2. 从内容区域提取风格特征
        3. 对上下区域进行 AI outpainting
        4. 使用深度图确保补全区域与内容区域的几何一致性
        5. 融合边缘, 平滑过渡
        """
        # 1. 创建 mask (黑色区域)
        mask = self._create_completion_mask(equirect_frame)

        # 2. 生成上下区域
        # 上部: 基于场景类型 (室内→天花板, 室外→天空)
        # 下部: 基于场景类型 (室内→地板, 室外→地面)
        prompt = self._analyze_scene(equirect_frame)

        # 3. SD inpaint
        completed = self._inpaint(equirect_frame, mask, prompt)

        return completed
```

## 7. 内存优化方案

### 7.1 问题分析

| 阶段 | 1080p 30s视频 (900帧) | 内存估算 |
|------|----------------------|---------|
| 帧加载 | 1920×1080×3×900 | ~5.6 GB |
| 深度图 | 1920×1080×4×900 | ~7.0 GB |
| 立体帧 | 1920×1080×3×900×2 | ~11.2 GB |
| SBS帧 | 7680×3840×3×900 | ~74.6 GB |
| **总计** | | **~98.4 GB** ❌ |

### 7.2 流式处理方案

```python
class StreamingPipeline:
    """流式处理: 一次只处理 N 帧, 不全部加载到内存。"""

    def __init__(self, buffer_size=16):
        self.buffer_size = buffer_size  # 每次处理 16 帧

    def process_stream(self, video_path, output_path):
        """
        处理流程:
        1. 逐帧读取 + 深度估计 (GPU 流水线)
        2. 逐帧生成立体视差
        3. 逐帧投影到等距柱状
        4. 直接通过 ffmpeg pipe 写入输出
        """
        cap = cv2.VideoCapture(video_path)
        encoder = ffmpeg.pipe(output_path)

        frame_idx = 0
        while True:
            ret, frame = cap.read()
            if not ret:
                break

            # 深度估计 (GPU上)
            depth = self.depth_estimator.estimate(frame)

            # 立体渲染
            left, right = self.stereo_renderer.render(frame, depth)

            # 等距投影
            sbs = self.eq_mapper.map_stereo_pair(left, right)

            # 直接写入编码器 (不保存到磁盘)
            encoder.write_frame(sbs)

            # 释放内存
            del depth, left, right, sbs

            frame_idx += 1

        encoder.close()
```

### 7.4 分块超分 (8K 输出)

```python
def upscale_tiled(frame, upscaler, scale=4, tile_size=512):
    """分块超分, 避免大分辨率导致 OOM。

    7680×3840 → 4× → 30720×15360 会 OOM
    分块处理: 每个 512×512 块独立超分后拼接
    """
    H, W = frame.shape[:2]
    out_H, out_H = H * scale, W * scale
    output = np.zeros((out_H, out_W, 3), dtype=np.uint8)

    for y in range(0, H, tile_size):
        for x in range(0, W, tile_size):
            tile = frame[y:y+tile_size, x:x+tile_size]
            tile_up = upscaler.upscale_frame(tile)
            output[y*scale:(y+tile_size)*scale,
                   x*scale:(x+tile_size)*scale] = tile_up

    return output
```
